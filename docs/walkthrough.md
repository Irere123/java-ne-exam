# End-to-end walkthrough

This walks the full billing lifecycle through the API using Swagger UI (or Postman).
It mirrors the [billing-lifecycle diagram](architecture.md#billing-lifecycle).

1. **Log in as admin** — `POST /api/auth/login` with the seed admin
   (`admin@java-exam.local` / `admin12345`). Copy the `accessToken` and click
   **Authorize** in Swagger to attach it to subsequent calls.

2. **Configure a tariff** — `POST /api/tariffs` (ADMIN). For example, a WATER tariff
   with a single flat tier:

   ```json
   {
     "meterType": "WATER",
     "serviceCharge": 2000,
     "vatRate": 18,
     "penaltyRate": 5,
     "effectiveFrom": "<this month>",
     "tiers": [{ "minUnits": 0, "maxUnits": null, "ratePerUnit": 323 }]
   }
   ```

3. **Register a customer** — `POST /api/customers` (ADMIN / OPERATOR).

4. **Register a meter** — `POST /api/meters` for that customer.

5. **Capture a reading** — `POST /api/readings` (OPERATOR). Consumption is derived from
   the previous and current readings.

6. **Generate the bill** — `POST /api/bills/generate?readingId=...` (FINANCE / ADMIN).
   A `BILL_GENERATED` notification is created by a database trigger — confirm it with
   `GET /api/notifications`.

7. **Approve the bill** — `POST /api/bills/{id}/approve`. The bill moves to `APPROVED`.

8. **Record payments** — `POST /api/payments` (a partial amount first, then the
   remainder). After a partial payment the bill is `PARTIALLY_PAID`; once the
   outstanding balance reaches zero it becomes `PAID` and a `PAYMENT_COMPLETED`
   notification is generated by the database trigger.

9. **View as the customer** — register/log in a `CUSTOMER` account with the **same
   email** as the customer record (the two are auto-linked via `customers.user_id`),
   then call `GET /api/me/bills`, `/api/me/payments`, and `/api/me/notifications`. An
   account with no linked profile gets a clear `404`.

## Applying late penalties (optional)

For an overdue, unpaid bill, `POST /api/bills/apply-penalties` (FINANCE / ADMIN) runs
the cursor-based stored procedure `sp_apply_late_penalties()`, which adds the tariff's
penalty and moves affected bills to `OVERDUE`. See
[Database routines](database-routines.md) for details.
